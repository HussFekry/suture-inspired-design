# Suture-Inspired Interface Design via FEA, Deep Learning, and Inverse Design

A computational pipeline for generating suture-inspired mechanical interfaces, evaluating their tensile-pullout behavior through finite-element analysis, with a planned extension to a learned surrogate model and inverse design.

---

## Overview

Biological sutures — the wavy interlocking joints found in skulls, seed pods, and ammonite shells — achieve remarkable combinations of strength, stiffness, and toughness through their interface geometry alone. This project builds an automated framework to:

1. **Generate** suture interface geometries from a unified parametric B-spline model
2. **Simulate** their tensile pullout response in Abaqus/Standard
3. **Extract** mechanical properties (peak force, stiffness, strain energy, peak stress)
4. **Learn** a forward geometry-to-property model *(planned)*
5. **Invert** the model to design geometries for target properties *(planned)*

The core novelty is a single **freeform B-spline generation method** that produces both known biological suture morphologies and unconstrained exploratory geometries, with mechanical performance emerging from simulation rather than being hand-tuned.

## 1 · Geometry Generation

All geometries are produced by one parametric B-spline framework, controlled by three variables:

| Variable | Meaning |
|----------|---------|
| `N` | number of control points (complexity) |
| `k` | spline order (smoothness) |
| `alpha` | amplitude fraction (interlocking depth) |

Four morphology families are generated by different control-point placement rules — three reproduce known biological suture forms, and a fourth samples the unconstrained design space beyond named morphologies:

| Jigsaw | Sinusoidal | Irregular | Freeform (unconstrained) |
|:------:|:----------:|:---------:|:------------------------:|
| ![jigsaw](https://github.com/HussFekry/suture-inspired-design/raw/main/Assests/jigsaw_00002_N11_k4.png) | ![sinusoidal](https://github.com/HussFekry/suture-inspired-design/raw/main/Assests/sinusoidal_00012_N14_k4.png) | ![irregular](https://github.com/HussFekry/suture-inspired-design/raw/main/Assests/irregular_00136_N14_k4.png) | ![freeform](https://github.com/HussFekry/suture-inspired-design/raw/main/Assests/Random%20freeform_00171_N11_k3.png) |
| bulb–neck teeth | wave interface | natural asymmetric | no imposed rule |

Each geometry is validated (no self-intersection, bounded curvature, clamped endpoints) and exported as (a) a 256×256 binary image for CNN input and (b) an Abaqus-ready JSON of curve points.

## 2 · Finite-Element Simulation

Each geometry is built in Abaqus/Standard as two interlocking halves and subjected to a **tensile pullout** test:

- Left edge fully fixed; right edge displaced +0.5 mm in tension
- Frictionless surface-to-surface contact, hard normal behavior
- CPS8R (8-node quadratic, plane-stress) elements to resolve the curved contact
- Geometric nonlinearity (NLGEOM) enabled

### Extracted properties

From each simulation's force–displacement response, four mechanical properties are recorded:

| Property | Meaning |
|----------|---------|
| `Fmax` | peak pullout force (strength) |
| `k_stiff` | initial stiffness (slope of the force–displacement curve) |
| `U_energy` | strain energy / toughness (area under the curve) |
| `sigma_max` | peak von Mises stress |

## Usage

### 1. Generate geometries (Colab)

Open `suture_pipeline.ipynb`, set the Drive path, and run the setup and dataset-generation sections. This produces the geometry images, a master CSV, and one JSON per geometry, then zips the JSONs for transfer.

### 2. Run the FEA batch (workstation with Abaqus)

Unzip the geometry JSONs into a `geometries/` folder alongside the scripts, then:

```bash
abaqus cae noGUI=run_batch_single.py
```

The runner processes every geometry, appends results to `results.csv`, logs progress to `batch_log.txt`, and **resumes automatically** if interrupted (already-completed geometries are skipped). Geometries that fail to converge are logged and skipped without stopping the batch.

Typical throughput is ~30–50 s per geometry; convergence rate is ~80%, which is expected for a diverse FEA dataset.

## Notes

Reported `sigma_max` values may exceed the material modulus due to stress singularities at sharp contact corners — a known FEA artifact. This is acceptable for comparative machine-learning use (all geometries are affected equally); absolute stress claims would use a smoothed or averaged measure.
